🧹 ProjectClean Pro

The most complete Unity project hygiene toolkit — scan, analyze, fix, and safely delete unused assets, missing references, duplicates, redundant packages, and bloated shaders across every Unity version and render pipeline.

v1.0.0 Unity 5.6 – 6.x Built-in · URP · HDRP Addressables Ready CI/CD Support

Overview

As Unity projects grow over months and years, they accumulate a silent weight of clutter: placeholder textures that were replaced, auto-generated materials never cleaned up, packages added for experiments that were abandoned, shader variants compiling for configurations that don't exist, and broken references left by deleted scripts. This slows down the editor, inflates build sizes, and makes onboarding new team members confusing.

ProjectClean Pro is a comprehensive project hygiene suite built natively for Unity 6's UI Toolkit with full backwards compatibility to Unity 5.6. It provides eight integrated analysis modules, a safety-first delete workflow with pre-delete archiving, full Addressables awareness, render-pipeline-specific shader analysis, and a batch CLI mode for automated CI/CD pipelines.

Key Design Principles

  • Safety first — Every delete operation previews results, confirms intent, and archives before removing. No surprise data loss.
  • Render-pipeline aware — Understands Built-in, URP, and HDRP asset types and won't flag pipeline-specific shaders as unused incorrectly.
  • Addressables aware — Treats Addressable groups as first-class dependency roots so Addressable-only assets are never falsely flagged as unused.
  • Non-destructive workflow — Archive mode moves files to a local .pcp_archive/ folder before deletion; restore with one click if you made a mistake.
  • Team-friendly — Ignore rules, exclusion filters, and configuration profiles can be committed to version control so the whole team shares the same setup.
  • CI/CD ready — Full command-line interface for integration into Unity Cloud Build, GitHub Actions, and any build pipeline.

Quick Start

Open Tools → ProjectClean Pro → Open Window. The dashboard shows six module tabs at the top. For a first-time full scan, click the Scan All button on the Dashboard tab. A full scan on a medium-sized project (5,000–15,000 assets) typically takes 15–45 seconds.

1
Run Full Scan
Dashboard → Scan All Modules. The progress bar shows which module is running. Results cache to disk — subsequent scans are 5–10× faster.
2
Review Results
Switch to each module tab. Each shows a sortable, filterable list of findings. Click any item to ping it in the Project window. Right-click for per-item options.
3
Configure Ignore Rules
Before deleting anything, set up your Ignore Rules (Settings tab) to exclude third-party folders, Resources, Streaming Assets, and any paths you want to always keep.
4

Select assets to delete. Click Preview Delete to see what would be removed. Click Archive & Delete to move to .pcp_archive/ and delete from the project. Restore any file from Archive View if needed.

Module 1 — Unused Asset Scanner

The core of ProjectClean Pro. Performs a complete dependency walk starting from every scene in Build Settings, every Prefab referenced from those scenes, every Resources/ folder (auto-included by Unity), every Addressables group, and every StreamingAssets/ path. Any asset not reachable from these roots is considered unused and reported.

How It Works

The scanner builds a full dependency graph using AssetDatabase.GetDependencies() with recursive mode enabled, then computes the set difference between all project assets and reachable assets. Results are grouped by asset type and folder path for readability.

Feature List

  • Build-settings-aware: Only scenes added to Build Settings are scanned as roots. You can optionally include all scenes in the project.
  • Resources folder awareness: All assets under any Resources/ folder are automatically treated as used (Unity includes them all at build time).
  • Addressables group support: Scans all Addressables groups as dependency roots. Assets only referenced from an Addressables group will NOT be falsely flagged as unused. (Requires Addressables package 1.16+)
  • AssetBundle support: AssetBundle manifests are respected — assets assigned to any AssetBundle are treated as used.
  • StreamingAssets pass-through: Files in StreamingAssets are always excluded (they can't be traced by the dependency graph).
  • Folder + type grouping: Results tree groups findings by folder first, then by type (Texture, Mesh, Material, Audio, Prefab, Script, etc.).
  • Uncompressed size column: Shows the uncompressed disk size of each unused asset so you can prioritize by impact.
  • Scene usage mapping: Hover or click any asset to see which scenes reference it (or confirm zero references).
  • Incremental scan cache: Asset modification timestamps are cached. Only changed assets are re-analyzed on subsequent scans.
  • Filter by type: Quickly filter results to show only textures, or only audio, or only scripts, etc.
  • Sort by size, name, path, type: Click column headers to sort.
  • Custom scan roots: Add additional folder paths or assets to treat as scan roots (e.g. a custom editor-only asset database).
  • Regex-based exclusions: Exclude assets matching path patterns (e.g. skip anything in Assets/ThirdParty/).

Understanding Results

Status BadgeMeaningRecommended Action
UNUSEDNot referenced from any scan rootReview and delete if confirmed unnecessary
MAYBE USEDReferenced only from a disabled/excluded sceneVerify before deleting

⚠️
Scripts note: The scanner has limited ability to detect unused C# scripts. Script references inside Prefabs and ScriptableObjects are detected, but dynamically loaded scripts (via Type.GetType(), reflection, or string-based AddComponent calls) cannot be traced. Always manually review flagged scripts before deleting.

Module 2 — Missing Reference Finder

Finds every broken reference in your project — the dreaded "Missing (Script)" and "None (Texture2D)" that silently break prefabs, ScriptableObjects, and scenes. This module reads the raw YAML/binary serialization of every asset to find GUIDs that point to non-existent assets.

Feature List

  • GUID-level scanning: Reads raw asset serialization (YAML mode) to find every GUID reference and validates each one exists in the AssetDatabase.
  • FileID + GUID cross-check: Validates both the GUID (file identity) and fileID (component identity within the file) for maximum accuracy.
  • Missing script detection: Finds GameObjects and Prefabs with detached MonoBehaviour components ("Missing (Script)" in inspector).
  • Missing texture / material / audio / prefab references: Validates all serialized UnityEngine.Object fields across every asset type.
  • Scene scanning: Scans every scene file, not just those in Build Settings.
  • Prefab scanning: Recursively scans nested prefabs.
  • ScriptableObject scanning: Validates all SO assets in the project.
  • Severity levels: Distinguishes between confirmed-broken (GUID absent) and ambiguous (GUID present but fileID absent — often internal Unity references).
  • Direct-navigate to broken field: Double-click any finding to open the containing asset and ping it in the inspector.
  • One-click null-out: For confirmed missing references, option to set the field to null programmatically (with undo support).
  • Binary asset support: Switches project to Force Text serialization temporarily for scanning if needed (with prompt and revert).

Severity Classification

SeverityTypeDescription
ERRORMissing FileID + GUIDBroken reference — the target asset no longer exists. Will cause runtime NullReferenceExceptions.
WARNINGMissing GUIDGUID not found in AssetDatabase. Likely deleted asset. Investigate.
INFOMissing FileID onlyUsually internal Unity references. Low risk but worth reviewing.

Module 3 — Duplicate Asset Detector

Finds duplicate assets across your project — not just by name, but by actual file content using SHA-256 content hashing. A texture imported twice, a mesh exported from Blender under two different names, or a sound effect copied into two packages: all are detected regardless of filename.

Feature List

  • Content-hash deduplication: Computes SHA-256 hash of the raw asset file bytes. Name and path are irrelevant — only the actual content determines duplicates.
  • Import-settings-aware: Optionally compares import settings alongside content, so the same texture with different compression settings is treated as distinct.
  • Type-aware grouping: Duplicates grouped by asset type for clarity.
  • Reference count per duplicate: Shows how many assets in the project reference each copy of the duplicate, so you can decide which copy to keep.
  • Merge assistant: For duplicate textures/meshes/audio, choose the "canonical" copy and the tool will automatically reroute all references from the duplicates to the canonical one before deleting the extras.
  • Material duplicate detection: Identifies materials with identical shader + property values even if stored as separate files.
  • Texture atlas cross-check: Detects textures that appear in sprite atlases and flags them as shared to prevent accidental deletion.
  • Hash cache: SHA-256 hashes are cached per-asset (invalidated by modification time), making repeat scans fast even on large projects.
  • Exclude patterns: Regex-based exclusions to skip intentional duplicates (e.g. platform-specific audio variants).

Module 4 — Interactive Dependency Graph

A visual, interactive graph showing how assets in your project reference each other. Essential for understanding the impact of a change or deletion before making it.

Feature List

  • Interactive canvas: Pan, zoom, and rearrange the graph nodes. Built with Unity's GraphView API (Unity 2020+) or a custom IMGUI canvas on older versions.
  • Per-asset investigation: Select any asset in the Project window, right-click → "Show in Dependency Graph" to center the graph on that asset and expand its upstream/downstream deps.
  • Depth control: Limit graph expansion to N levels of depth (default 2) to keep large graphs manageable.
  • Upstream view (who uses this?): Show all assets that depend on a selected asset.
  • Downstream view (what does this use?): Show all assets that a selected asset depends on.
  • Circular dependency detection: Highlights circular reference chains in red — a common source of build and editor performance issues.
  • Node coloring by type: Textures, Meshes, Scripts, Materials, Prefabs, Scenes, and Audio each have distinct colors.
  • Node size by file size: Optionally scale node size by uncompressed asset size to visually identify the heaviest assets.
  • Export graph as SVG/PNG: Save the current graph view for documentation or team communication.
  • Filter by folder: Limit the graph to assets within a specific folder to reduce visual complexity.
  • Orphan highlighting: Assets with zero incoming edges (no referrers) are highlighted — these are deletion candidates.

Module 5 — Package Auditor

Analyzes your project's installed UPM packages and built-in packages to identify ones that are not referenced by any code or asset in the project. Unused packages add compilation overhead, inflate build size, and increase the risk of API conflicts.

Feature List

  • UPM package scan: Reads Packages/manifest.json and analyzes all installed packages.
  • Usage detection: Checks for using directives, component references, asmdef references, and inspector-assigned references that depend on each package.
  • Built-in package detection: Lists all enabled built-in packages (e.g. Legacy Particle System, Terrain Tools) and flags ones with zero project usage.
  • Dependency chain awareness: Won't flag Package A as unused if it's a required dependency of Package B that IS in use.
  • Safe removal guidance: For each flagged package, provides the Package Manager command and warns if removal would trigger API compilation errors.
  • Package size reporting: Shows disk footprint of each package so you can prioritize removals by impact.
  • One-click removal: Trigger package removal directly from the ProjectClean Pro UI (calls PackageManager API under the hood).
  • Package health check: Flags packages not compatible with your current Unity version, packages with available updates, and deprecated packages.
  • Whitelist: Mark packages as "always keep" to suppress findings for intentionally installed but currently unused packages (e.g. test frameworks).
⚠️
Caution: Removing packages can cause compilation errors if any code uses their APIs. Always use the Preview Impact button before removing a package. The tool will perform a dry-run compilation check and report any errors before any changes are made.

Module 6 — Shader & Shader Variant Analyzer

Shader variant compilation is one of the most significant and least understood sources of build time bloat in Unity projects. This module provides deep visibility into your shader usage across all render pipelines.

Feature List

  • Shader inventory: Lists every shader asset in the project with its type (Surface, Unlit, Shader Graph, HLSL Custom), render pipeline target, and estimated variant count.
  • Material-to-shader mapping: For each shader, shows every material in the project that uses it. Shaders with zero material users are flagged.
  • Render pipeline correctness: Detects shaders targeting the wrong pipeline — e.g. a Built-in Standard shader used in a URP project (wastes variant compilation budget and may render incorrectly).
  • Shader Graph analysis (Unity 2021+): Identifies Shader Graph assets that are never used by any material, and flags Shader Graphs with excessive node counts that may slow editor compilation.
  • Variant collection audit: Lists all ShaderVariantCollection assets and checks for stale entries (variants referenced in the collection but whose keyword combinations no longer exist in the shader).
  • Keyword redundancy check: Identifies shader keywords that are defined but never enabled by any material in the project, which can safely be stripped.
  • Shader compilation error detection: Lists all shaders with compile errors on the current platform and render pipeline target.
  • URP Shader Graph upgrader integration: Detects legacy Built-in shaders that have official URP/HDRP upgrade paths and links to Unity's upgrade utilities.
  • HDRP material validator integration: Surfaces HDRP material validation results (normally only shown in the Scene view) in the ProjectClean Pro window for batch review.
  • Estimated variant count: Uses static analysis of #pragma multi_compile and #pragma shader_feature directives to estimate the total variant combinatorial count per shader.

Module 7 — Project Size Profiler

A detailed breakdown of your project's disk footprint by asset type, folder, and individual asset — with build-size estimates and compression ratio analysis.

Feature List

  • Total project size by type: Visual breakdown (treemap and bar chart) of disk usage across Textures, Meshes, Audio, Animations, Scripts, Materials, etc.
  • Folder drill-down: Click any folder in the treemap to zoom in and see its contents by size.
  • Estimated build size contribution: Uses Unity's compression estimates to project how much each asset type contributes to your final build, per target platform.
  • Texture compression audit: Identifies textures with suboptimal compression settings for your target platform (e.g. uncompressed RGBA32 textures on mobile).
  • Oversized texture detection: Flags textures with a source resolution larger than their Max Texture Size import setting — wasted source file disk space.
  • Audio compression audit: Identifies uncompressed PCM audio clips that could use Compressed In Memory or Streaming to reduce build size.
  • Mesh complexity report: Lists meshes by vertex count, poly count, and whether Read/Write is enabled (which doubles memory usage) unnecessarily.
  • Platform-specific size estimates: Switch between Android, iOS, PC, WebGL, and Console target platforms to see how size estimates change.
  • Historical comparison: Save size snapshots and compare against them to track growth over time.
  • Optimization recommendations: For each category, shows Unity-doc-linked recommendations for reducing size (e.g. "Enable Crunch compression for these 14 textures to save ~28MB").

Module 8 — Safe Delete System

Every deletion in ProjectClean Pro goes through the Safe Delete system. This is not optional — it's the only way to delete. The system is designed to make cleanup feel confident, not terrifying.

Feature List

  • Pre-delete preview: Before any deletion, shows a full list of assets to be deleted with their sizes, types, and current reference counts. Includes a confirmation prompt.
  • Archive before delete: All files are first moved to .pcp_archive/YYYY-MM-DD_HH-MM-SS/ at the project root. The archive preserves the original folder structure.
  • Post-delete undo: The archive view shows all previous deletion sessions. Select any session and click Restore Session to move all archived assets back to their original paths and re-register them with the AssetDatabase.
  • Selective restore: Restore individual files from an archive session, not just the entire batch.
  • Meta file handling: Automatically deletes corresponding .meta files with every asset to prevent orphan meta issues.
  • Reference null-out option: Optionally null-out all serialized references to deleted assets before deletion, preventing missing-reference warnings on affected Prefabs and Scenes.
  • Git-aware mode: If the project is a Git repository, uses git rm for tracked files instead of direct deletion, preserving Git history.
  • Dry-run mode: Run the delete flow without actually deleting anything — logs what would be deleted to the Console.
  • Archive size management: Settings let you auto-purge archive sessions older than N days or larger than N MB.
  • Delete log: All deletion operations are logged to Library/ProjectCleanPro/deletions.log with timestamps, asset paths, sizes, and the user who triggered the deletion.
💡
Team Workflow Tip: Add .pcp_archive/ to your .gitignore. Each developer's local archive is their own safety net. Commit the delete log (Library/ProjectCleanPro/deletions.log) to track what was cleaned up and when.

Addressables Support

ProjectClean Pro has first-class support for the Addressables system. If Addressables is installed in your project, the scanner automatically detects it and adjusts all analysis accordingly.

How Addressables Integration Works

  • Group scanning: All assets in all Addressables groups (Default, Remote, Local) are treated as dependency roots — they and all their transitive dependencies are marked as "in use."
  • AssetReference detection: Scans MonoBehaviour and ScriptableObject fields for AssetReference typed properties and follows their GUIDs as dependency roots.
  • Labels support: Assets with Addressables labels are recognized as used regardless of whether they appear in any scene.
  • Addressables-only asset detection: Correctly identifies assets that are ONLY used via Addressables (not referenced from any scene) and does not flag them as unused.
  • Group cleanliness audit: Reports Addressables groups that contain assets also directly embedded in scenes — a common source of duplicate builds.
  • Missing Addressables reference detection: Finds AssetReference fields whose GUID no longer exists in the project, which would fail at runtime with a silent null.
  • Stale group entries: Detects assets added to an Addressables group but subsequently deleted from the project (the group entry remains but the asset is gone).
ℹ️
Version requirement: Addressables integration requires Addressables package version 1.16.0 or higher. ProjectClean Pro detects the installed version via PackageInfo and gracefully degrades on older versions.

Scripting API

All core functionality is available via a public C# API for custom editor scripts, build processors, and CI automation.

Namespace

using ProjectCleanPro.Editor;

PCPScanner — Running a Scan

Run full scan and get results
// Synchronous scan (blocks editor, best for CI)
PCPScanResult result = PCPScanner.RunFullScan(new PCPScanOptions
{
    IncludeAllScenes        = false,  // Only Build Settings scenes
    IncludeAddressables     = true,
    IncludeAssetBundles     = true,
    IncludeResourcesFolders = true,
    ScanForDuplicates       = true,
    ScanForMissingRefs      = true,
    AnalyzeShaders          = true,
    AuditPackages           = true,
});

// Check results
Debug.Log($"Unused assets: {result.UnusedAssets.Count}");
Debug.Log($"Missing references: {result.MissingReferences.Count}");
Debug.Log($"Duplicates: {result.DuplicateGroups.Count}");

// Async scan (non-blocking, best for editor workflows)
await PCPScanner.RunFullScanAsync(options, onProgress: (progress, label) =>
{
    EditorUtility.DisplayProgressBar("Scanning...", label, progress);
});

PCPScanResult — Working with Results

foreach (PCPUnusedAsset asset in result.UnusedAssets)
{
    Debug.Log($"{asset.AssetPath} | {asset.Type} | {asset.UncompressedSizeBytes} bytes");
}

foreach (PCPMissingReference missingRef in result.MissingReferences)
{
    Debug.Log($"{missingRef.ContainerPath} → field '{missingRef.FieldName}' " +
              $"severity: {missingRef.Severity}");
}

foreach (PCPDuplicateGroup group in result.DuplicateGroups)
{
    Debug.Log($"Duplicate group ({group.Hash}) — {group.Copies.Count} copies, " +
              $"{group.WastedBytes} bytes wasted");
}

PCPSafeDelete — Programmatic Deletion

// Build a list of assets to delete
var toDelete = result.UnusedAssets
    .Where(a => a.Type == PCPAssetType.Texture && a.UncompressedSizeBytes > 1_000_000)
    .Select(a => a.AssetPath)
    .ToList();

// Preview (dry run, returns what would be deleted)
PCPDeletePreview preview = PCPSafeDelete.Preview(toDelete);
Debug.Log($"Would delete {preview.AssetCount} assets ({preview.TotalBytes} bytes)");

// Execute safe delete (archives first)
PCPDeleteResult deleteResult = PCPSafeDelete.Execute(toDelete, new PCPDeleteOptions
{
    ArchiveBeforeDelete = true,  // Always recommended
    NullOutReferences   = false,
    UseGitRm            = PCPGitUtils.IsGitRepository(),
});

Debug.Log($"Deleted {deleteResult.DeletedCount} assets. " +
          $"Archive session: {deleteResult.ArchiveSessionId}");

PCPReportExporter — Generating Reports

// Export JSON report
PCPReportExporter.ExportJSON(result, "Assets/../ProjectCleanReport.json");

// Export CSV (useful for spreadsheet review with teams)
PCPReportExporter.ExportCSV(result, "Assets/../ProjectCleanReport.csv",
    PCPReportScope.UnusedAssets | PCPReportScope.MissingReferences);

// Export HTML report (shareable, human-readable)
PCPReportExporter.ExportHTML(result, "Assets/../ProjectCleanReport.html");

PCPIgnoreRules — Managing Ignore Rules from Code

var settings = PCPSettings.Load();

// Add ignore rules programmatically
settings.IgnoreRules.Add(new PCPIgnoreRule
{
    Type       = PCPIgnoreType.PathPrefix,
    Pattern    = "Assets/ThirdParty/",
    Comment    = "Third-party assets — managed externally",
});

settings.IgnoreRules.Add(new PCPIgnoreRule
{
    Type    = PCPIgnoreType.Regex,
    Pattern = @"Assets/.*_BACKUP_.*",
    Comment = "Backup files",

settings.Save();

CLI & CI/CD Integration

ProjectClean Pro includes a command-line interface that runs as a Unity batch-mode process. This allows you to integrate project scanning into any automated build or CI pipeline.

Basic CLI Usage

Shell / Terminal

"C:/Program Files/Unity/Hub/Editor/6000.0.0f1/Editor/Unity.exe" \
  -batchmode \
  -quit \
  -projectPath "C:/Projects/MyGame" \
  -executeMethod ProjectCleanPro.Editor.CLI.PCPCommandLine.RunScan \
  -pcp-output "Reports/pcp_report.json" \
  -pcp-format json \
  -pcp-modules all

# Run scan, fail the build if unused assets > 50MB total
"C:/Program Files/.../Unity.exe" \
  -batchmode -quit \
  -projectPath "C:/Projects/MyGame" \

Ignore Rules System

Ignore rules tell ProjectClean Pro which assets to skip entirely — they won't appear in any scan results. Rules are stored in ProjectSettings/PCPSettings.asset (a ScriptableObject) so they can be version-controlled and shared across the team.

Rule Types

Quick-Add via Right-Click

In any result list, right-click an asset → "Add to Ignore Rules" to instantly add an exact-path ignore rule for that asset. Right-click a folder → "Ignore this folder and its contents" to add a folder rule.

The pcp-keep AssetDatabase Label

You can tag any asset with the AssetDatabase label pcp-keep directly in the Unity inspector (Asset Labels section) to permanently exclude it from all ProjectClean Pro results without creating a rule. Useful for assets that are truly never referenced but should never be deleted (e.g. template files, CI test fixtures).

FAQ

Will ProjectClean Pro delete my assets automatically?

Never without explicit user action. The tool only analyzes and reports. All deletions require you to manually select assets and click through the Safe Delete workflow with a preview step. Even then, files are archived before being removed from the project.

Why are some assets marked as UNUSED when I know I use them?

The most common causes: (1) The scene they're used in is not added to Build Settings and you haven't enabled "Include All Scenes" mode. (2) The asset is loaded at runtime via Resources.Load() — add the folder to a custom scan root or ensure it's under a Resources/ folder. (3) The asset is loaded dynamically via path string or reflection — add an ignore rule or pcp-keep label.

Does it support Streaming Assets?

Assets in Assets/StreamingAssets/ are automatically excluded from all scans. They're not traceable by the AssetDatabase dependency system since they're accessed by path at runtime, so flagging them would produce false positives.

Can I use it in a team environment?

Yes. Settings are stored in ProjectSettings/PCPSettings.asset which can be committed to version control. This ensures the whole team uses the same ignore rules and scan configuration. The archive folder should be gitignored.

Does it work with Unity's new Input System / Cinemachine / etc.?

Yes. ProjectClean Pro works at the AssetDatabase level and is agnostic to which Unity packages you use. The Package Auditor module understands package dependency chains, so it won't flag Cinemachine as unused if your project depends on it transitively.

How does it handle Editor-only assets?

Editor-only assets (under Editor/ folders or with platform constraints) are scanned separately from runtime assets. The results panel has a toggle to show/hide Editor-only assets to keep the runtime-asset list focused.

Will it find unused animation parameters or empty Animator states?

Not in v1.0. Animator graph cleanup is planned for a future module. Currently the tool finds unused AnimationClip assets (clips not referenced by any Animator Controller or directly) but does not inspect the Animator graph itself.

Known Limitations

🚫
Reflect-loaded scripts: Scripts dynamically loaded via Assembly.GetType(), Type.GetType(), or string-based AddComponent calls cannot be traced. Review all flagged scripts manually before deletion.
⚠️
Procedurally loaded assets: Assets loaded via AssetDatabase.LoadAssetAtPath() at runtime or in custom build processors may not be traced unless you add them as custom scan roots.
⚠️
Third-party build pipeline overrides: Tools that override Unity's build pipeline (e.g. VRChat SDK, some console platform SDKs) may inject references that ProjectClean Pro cannot trace. Use the Ignore Rules system to protect any assets their documentation says to keep.
⚠️
Binary serialization mode: Projects using Binary serialization mode (not the default) require switching to Force Text for Missing Reference scanning. ProjectClean Pro prompts before doing this and reverts immediately after the scan.
  • Animator Controller internal graph analysis (unused states, dead parameters) — planned for v1.1
  • Playable Graph and Timeline asset analysis — planned for v1.2
  • Localization package integration (unused locale assets) — planned for v1.2
  • Custom SRP shader pipeline detection is heuristic-based and may need manual configuration
  • Large projects (50,000+ assets) may see scan times of 2–5 minutes on first run. Subsequent cached scans are significantly faster.
